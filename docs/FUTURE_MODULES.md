# Будущие модули

## Обзор

Этот документ описывает план развития других разделов платформы (детские сады, университеты, курсы). Пока мы фокусируемся на школах, но фундамент для других модулей уже заложен.

## Структура

Каждый модуль будет следовать той же структуре, что и модуль школ:

```
src/app/{module}/
├── layout.tsx                    # Layout с {Module}Header
├── page.tsx                      # Главная страница
├── list/
│   └── page.tsx                 # Каталог
└── [slug]/
    ├── page.tsx                 # Профиль
    └── not-found.tsx            # 404

src/components/{module}/Header/
├── {Module}Header.tsx           # Главный компонент
├── {Module}TopBar.tsx           # Верхняя строка
└── {Module}MainNav.tsx          # Основная навигация
```

## Общий подход

Все модули будут использовать:

1. **BaseHeader** - базовая логика sticky header и скролла
2. **Общие компоненты** - `RegionSelector`, карточки, фильтры (адаптированные)
3. **Единый стиль** - дизайн-система платформы
4. **SEO оптимизация** - метаданные для каждого модуля
5. **RegionProvider** - работа с регионами/районами

## Когда начинать разработку

После завершения основных функций модуля школ:

1. ✅ Миграция архитектуры (завершено)
2. ✅ Базовые компоненты Header (завершено)
3. ⏳ Полная функциональность школ
4. ⏳ Стабильная работа школ
5. ⏳ Затем - следующий модуль

## Рекомендации

1. **Не копировать код школ слепо** - каждый модуль имеет свои особенности
2. **Использовать BaseHeader** - это основа для всех модулей
3. **Адаптировать общие компоненты** - не создавать с нуля
4. **Тестировать модульность** - модули не должны конфликтовать друг с другом
