# UI/UX: Админка — Управление школами

> **Раздел**: Админ-панель / Личный кабинет школы
> **Версия**: 1.0
> **Дата**: Декабрь 2024

---

## 1. Концепция

### Философия интерфейса

```
НЕ одна огромная форма на 100+ полей, а:

┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│   1. БЫСТРОЕ СОЗДАНИЕ          2. ПОСТЕПЕННОЕ ОБОГАЩЕНИЕ       │
│   ──────────────────           ──────────────────────────       │
│   Минимум данных               Разделы по категориям            │
│   за 2 минуты                  заполняются постепенно           │
│   ↓                            ↓                                │
│   Школа появляется             Больше данных = выше рейтинг     │
│   в каталоге                   + лучше позиция                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Прогрессивное заполнение** | Начать легко, углублять постепенно |
| **Визуальный прогресс** | Видно что заполнено, что нет |
| **Мотивация через рейтинг** | Показываем как данные влияют на рейтинг |
| **Единый интерфейс** | Одинаково для владельцев и админов |
| **Автосохранение** | Данные не теряются |

## 2. Структура разделов

### Иерархия данных

```
ПРОФИЛЬ ШКОЛЫ
│
├── 🟢 УРОВЕНЬ 1: Обязательные (для публикации)
│   └── Базовая информация
│       • Название, тип, описание
│       • Контакты, адрес
│       • Классы, язык, цена
│       └── ~15 полей → 2-3 минуты
│
├── 🟡 УРОВЕНЬ 2: Важные (влияют на рейтинг)
│   ├── Образовательная программа
│   ├── Педагогический состав
│   ├── Инфраструктура
│   └── Услуги и сервисы
│       └── ~40 полей → 10-15 минут
│
├── 🔵 УРОВЕНЬ 3: Полные (максимальный рейтинг)
│   ├── Результаты и достижения
│   ├── Поступление и приём
│   ├── Финансы (детально)
│   └── Документы и лицензии
│       └── ~30 полей → 10 минут
│
└── 📸 МЕДИА
    ├── Фотографии
    ├── Видео
    └── Виртуальный тур
```

### Сводная таблица разделов

| # | Раздел | Полей | Уровень | Влияние на рейтинг |
|---|--------|-------|---------|-------------------|
| 1 | Базовая информация | ~15 | 🟢 Обязательный | Минимальное |
| 2 | Образовательная программа | ~20 | 🟡 Важный | +10 баллов |
| 3 | Педагогический состав | ~15 | 🟡 Важный | +8 баллов |
| 4 | Инфраструктура | ~25 | 🟡 Важный | +5 баллов |
| 5 | Услуги и сервисы | ~10 | 🟡 Важный | +2 балла |
| 6 | Результаты и достижения | ~20 | 🔵 Полный | +15 баллов |
| 7 | Поступление и приём | ~15 | 🔵 Полный | +2 балла |
| 8 | Финансы (подробно) | ~15 | 🔵 Полный | +1 балл |
| 9 | Документы и лицензии | ~8 | 🔵 Полный | +2 балла |
| 10 | Фотографии | ~10 фото | 📸 Медиа | +1 балл |
| 11 | Видео | ~2 видео | 📸 Медиа | +0.5 балла |

## 9. Технические требования

### API Endpoints

```typescript
// Школы
GET    /api/admin/schools              // Список своих школ
POST   /api/admin/schools              // Создать школу
GET    /api/admin/schools/:id          // Получить школу
PATCH  /api/admin/schools/:id          // Обновить школу
DELETE /api/admin/schools/:id          // Удалить школу

// Разделы школы
GET    /api/admin/schools/:id/sections           // Все разделы с прогрессом
GET    /api/admin/schools/:id/sections/:section  // Данные раздела
PATCH  /api/admin/schools/:id/sections/:section  // Обновить раздел

// Медиа
POST   /api/admin/schools/:id/photos   // Загрузить фото
DELETE /api/admin/schools/:id/photos/:photoId
PATCH  /api/admin/schools/:id/photos/:photoId  // Изменить категорию/обложку

// Статусы
POST   /api/admin/schools/:id/submit   // Отправить на модерацию
POST   /api/admin/schools/:id/publish  // Опубликовать (только админ)
POST   /api/admin/schools/:id/reject   // Отклонить (только админ)
```

### Типы данных

```typescript
interface School {
  id: string;
  status: 'draft' | 'pending' | 'published' | 'rejected' | 'suspended';
  completeness: number; // 0-100
  
  // Базовая информация
  basic: BasicInfo;
  
  // Разделы
  sections: {
    education: EducationSection;
    teachers: TeachersSection;
    infrastructure: InfrastructureSection;
    services: ServicesSection;
    results: ResultsSection;
    admission: AdmissionSection;
    finance: FinanceSection;
    documents: DocumentsSection;
  };
  
  // Медиа
  media: {
    photos: Photo[];
    videos: Video[];
  };
  
  // Мета
  createdAt: Date;
  updatedAt: Date;
  publishedAt?: Date;
}

interface SectionProgress {
  section: string;
  completeness: number;
```
